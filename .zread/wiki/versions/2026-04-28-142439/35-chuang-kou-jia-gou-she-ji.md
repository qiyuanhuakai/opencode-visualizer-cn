本文档阐述 vis.thirdend 应用的多窗口架构设计，涵盖浮动窗、面板、工具窗等核心概念及其状态管理机制。该架构支持实时流式输出、异步内容渲染和灵活的布局管理，是整个 IDE 功能实现的基础。

## 核心架构概念
窗口系统由三大核心组件构成：浮动窗（Floating Window）用于临时展示流式输出和临时内容；停靠面板（Dock Panel）包括侧边栏（SidePanel）、顶栏（TopPanel）等固定区域；工具窗口（Tool Window）提供特定功能的专用界面，如终端、代码编辑等。

所有窗口通过统一的入口 `FloatingWindow.vue` 进行渲染，该组件负责处理滚动、搜索、主题等通用逻辑，并通过 composables 实现与状态管理的解耦。

## 状态管理层级
窗口状态管理采用分层架构：全局窗口注册表由 `useFloatingWindows` 维护，提供窗口的创建、销毁、内容更新等核心操作；单个窗口状态由 `useFloatingWindow` 封装，暴露聚焦、最小化、打开等 API；渲染状态由 `useRenderState` 跟踪，记录加载、错误、完成等生命周期状态。

这种分层设计使得状态变更可追溯、可测试，同时支持并发窗口操作。

## 浮动窗生命周期
浮动窗从创建到销毁经历完整的状态流转：初始状态 → 内容接收 → 渲染中 → 完成/错误 → 用户交互 → 关闭。每个状态通过响应式数据驱动 UI 更新，并通过 `notifyContentChange` 机制通知滚动管理器处理流式追加内容。

源码中通过 `watch` 监听 `resolvedHtml` 变化实现智能滚动位置保留：当处于手动模式或跟随模式下用户已手动滚动时，内容更新后自动恢复滚动位置。

## 流式渲染管线
流式数据通过 SSE 连接到达后，经 `sseConnection.ts` 解析为消息块，由 `useRenderState` 协调渲染流程。渲染过程中，`CodeContent.vue` 组件负责将 Markdown/代码块转换为高亮 HTML，并通过 `workerRenderer.ts` 在 Web Worker 中执行繁重的语法分析任务以避免阻塞主线程。

渲染完成的 HTML 存储在窗口 entry 的 `resolvedHtml` 字段中，触发浮动窗的 `watch` 更新。

## 搜索与导航系统
内置的内容搜索功能通过 `useContentSearch` 实现，支持正则表达式匹配、结果计数、当前位置高亮。搜索框集成在浮动窗头部，支持键盘快捷键导航（Enter/Shift+Enter）。搜索结果实时计算，与内容渲染解耦，确保大数据量下的响应性能。

## 主题与样式隔离
每个浮动窗支持独立主题配置，通过 `floatingWindowTheme.ts` 定义主题类型（跟随应用、暗色、亮色），并使用 CSS 变量实现动态切换。`ThemeInjector.vue` 负责将主题变量注入到 Shadow DOM 或全局作用域，确保渲染内容样式一致性。

## 工具窗口架构
工具窗口采用统一的基础组件 `ToolWindow/Default.vue`，具体功能通过插件式组件实现：`Bash.vue` 提供终端功能，`Read.vue` 处理文件阅读，`Edit.vue` 集成代码编辑能力。每个工具窗通过 `utils.ts` 共享状态逻辑，包括会话选择、权限检查、进度跟踪。

## 布局管理策略
应用采用弹性网格布局，主区域显示会话内容，侧边栏展示文件树和工具面板。浮动窗以绝对定位浮动于主区域之上，通过 z-index 管理堆叠顺序。最小化状态将窗口收缩为标题栏，状态栏显示窗口计数和活动状态。

## 并发与性能优化
窗口渲染使用 Web Worker 池化技术，语法高亮、Markdown 解析等 CPU 密集型任务在后台线程执行。主线程仅负责 DOM 更新，通过 `requestIdleCallback` 调度低优先级渲染任务。滚动同步采用防抖策略，避免频繁计算导致的卡顿。

## 事件流架构
全局事件通过 `useGlobalEvents` 统一管理，支持窗口间通信、快捷键分发、状态广播。事件采用发布-订阅模式，支持命名空间隔离，避免不同功能模块的事件冲突。

## 可扩展性设计
架构通过依赖注入和组合式 API 实现高度可扩展：新窗口类型只需实现对应的 `useWindow` composable 并注册到全局管理器；自定义渲染器通过 `renderers/` 目录下的组件扩展；主题系统支持插件式主题注册。
